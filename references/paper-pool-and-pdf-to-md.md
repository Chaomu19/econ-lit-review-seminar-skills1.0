# 论文池构建与 PDF 转 Markdown 工作流

## 目标

构建可信的文献综述论文池，并将确认无误的 PDF 转换为 Markdown，使后续 AI 能更高效地引用、分组和总结证据。

## 第一步：多模型审阅论文池

使用 2-3 个可信 AI 模型或研究工具，分别提出、审阅和质疑论文池。

要求每个模型检查：

- 论文是否匹配组会主题。
- 是否遗漏经典基础文献。
- 是否过度偏向某个新近研究角度。
- 是否覆盖理论、实证、政策、制度机制等必要维度。
- 这组论文能否支撑一篇连贯的文献综述，而不是松散书目。

必须人工核验：

- 题名。
- 作者。
- 年份。
- 期刊或工作论文来源。
- 研究问题。
- 与综述主题的相关性。

## 第二步：论文下载策略

自动化爬取和下载论文默认不可靠。在原项目经验中，自动下载正确率大约只有 `2/27`。

默认策略：

- 不承诺 AI 可以自动下载正确论文。
- 如果尝试自动下载，应将结果视为“未核验”。
- 用户或可信工具必须确认每个 PDF 后，才把它放入工作文件夹。
- 不要把 PDF 提交到 GitHub。

## 第三步：整理已确认 PDF

将确认无误的 PDF 放入同一个本地工作文件夹。推荐结构：

```text
papers/
├── 01_value_arbitrage_capacity/
├── 02_welfare_environment_system_cost/
├── 03_market_power_competition/
├── 04_market_design_policy_investment/
└── 05_local_context_or_application/
```

具体板块名称应根据综述主题调整。这个结构只是工作约定，不是固定分类法。

## 第四步：PDF 转 Markdown

必须保留这一步。转换工具不应依赖用户的个人 API 或私人账号。

Markdown 输出应尽可能保留：

- 论文题名、作者、年份、期刊。
- 章节标题。
- 摘要和结论。
- 公式和模型描述。
- 表格与表题。
- 图题、图注和图片路径。
- 原文中的 `Fig.` 与 `Table` 编号。

推荐输出结构：

```text
converted_md/
├── 01_value_arbitrage_capacity/
│   └── Author_Year_Title/
│       ├── full.md
│       └── images/
└── ...
```

## 第五步：转换质量检查

对每篇论文快速检查：

- `full.md` 是否对应正确 PDF。
- 题名、作者和年份是否正确。
- 主要章节标题是否完整。
- 关键图表是否可见或可引用。
- 公式是否足够可读，能否用于解释。
- 图片路径相对 Markdown 文件是否稳定。

如果转换质量较差，应重新转换，或标记该论文需要直接检查 PDF。

## GitHub 边界

不要纳入：

- 原始 PDF。
- 从出版商 PDF 中提取的大量图片，除非版权与引用边界明确。
- API 密钥或私人转换凭据。
- 默认生成的大型转换目录。

通常更适合上传模板和工作流说明，让每个用户在本地运行转换。
