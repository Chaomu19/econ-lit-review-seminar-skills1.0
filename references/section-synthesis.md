# 研究板块综合工作流

## 目标

把多篇论文压缩为一个可复述、可放入 PPT 的研究板块认知。

## 单篇论文抽取项

对每篇论文抽取：

- `研究问题`：论文到底在问什么？
- `方法`：理论模型、优化模型、仿真、计量识别、制度比较、综述等。
- `关键结论`：论文真正证明或展示了什么？
- `标签`：4-6 个便于记忆的关键词。
- `PPT 去向`：总览表、精讲页、图表页、争议页、备份页，或不进入 PPT。

## 板块级输出

```md
# 板块名称

## 主线目标

- 核心问题:
- 与前后板块的关系:

## 文献整理

| 文献 | 优先级 | 研究问题 | 方法 | 关键结论 | PPT 去向 |
|---|---|---|---|---|---|

## 建议讲述顺序

1. ...
2. ...
3. ...

## 主题对照 / 争议对照 / 因果链

- ...

## 最值得记住的 3 条结论

1. ...
2. ...
3. ...

## 100-150 字口播稿

...
```

## 质量标准

板块综合应帮助用户在组会上自然表达，而不是机械罗列摘要。

优先：

- 跨文献关系。
- 对比、条件和分歧。
- 机制链条。
- 清晰的过渡句。

避免：

- 过长摘要。
- 重复论文 abstract。
- 把所有论文看得一样重要。
