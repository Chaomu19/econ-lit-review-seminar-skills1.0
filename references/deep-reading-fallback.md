# 兜底精读工作流

## 目标

只为组会中可能被追问的论文准备更深入的笔记。

这不是每篇论文的默认流程。

## 使用场景

在以下情况下使用兜底精读：

- 某篇论文有 2-3 页精讲 PPT。
- 幻灯片展示了公式、模型假设或核心图表。
- 该论文结论反直觉或容易引发争议。
- 用户觉得页面可以展示，但自己还讲不透。
- 老师可能追问数据、识别策略、机制或外部有效性。

## 输出结构

```md
# Author Year: 兜底精读

## 1. 这篇论文为什么会出现在 PPT 中

## 2. 论文真正回答的研究问题

## 3. 模型 / 方法 / 识别策略

## 4. PPT 中使用的图表

对每个图表：

- 原文编号:
- 展示了什么:
- 关键模式:
- 经济学解释:
- 可能追问:
- 简短回答:

## 5. 核心发现与局限

## 6. 可能问答

## 7. 60-90 秒口播解释
```

## 规则

- 聚焦 PPT 中展示的页面和论断。
- 除非用户要求，不要重写完整论文摘要。
- 如果公式和方法是核心，应放在结论之前。
- 对不确定细节标记 `需要复核`。
