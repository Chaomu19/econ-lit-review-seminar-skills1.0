# 大纲、分组与精讲层级识别工作流

## 目标

在撰写详细笔记之前，先利用转换后的 Markdown 构建组会汇报的学术结构。

该步骤需要回答：

- 文献综述的核心问题是什么？
- 应按哪些研究板块组织论文？
- 哪些论文应精讲，哪些属于重点阅读，哪些只作为补充阅读？
- 每篇论文服务于 PPT 的哪一部分？

## 推荐流程

1. 从转换后的 Markdown 中扫描论文元信息、摘要、引言、方法、图表和结论。
2. 根据研究问题和机制提出 4-6 个研究板块，而不是只按年份排序。
3. 对每个板块总结：
   - 核心研究问题。
   - 常用研究方法。
   - 主要论文。
   - 板块内部的分歧、争议或张力。
   - 与下一板块的过渡关系。
4. 为每篇论文分配优先级：
   - `精讲`：支撑综述主论证，通常会获得 2-3 页 PPT。
   - `重点阅读`：支撑板块判断，可出现在综述表或综合页。
   - `补充阅读`：用于背景、备份或问答。
5. 让用户先审阅和调整分组，再进入 `PPT完整内容.md` 生成。

## 好分组的判断标准

好的研究板块应当：

- 由问题或机制驱动。
- 各板块相对均衡，不让一个板块吞掉全部综述。
- 排序形成论证链条，而不是简单文件夹列表。
- 能自然生成过渡句。
- 对 PPT 页序规划有帮助。

## 避免

- 按论文逐篇罗列，缺少综合。
- 把所有论文都标为精讲。
- 把用户个人时间计划当作综述结构。
- 在综述逻辑稳定前生成大量长篇笔记。

## 最终交付物

生成一份组会大纲，包括：

- 汇报标题。
- 核心命题。
- 研究板块顺序。
- 论文优先级表。
- 建议 PPT 页序。
- 可能需要兜底精读的论文。
